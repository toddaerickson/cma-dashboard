# CMA Dashboard — Barbell PP + GEP Credit Overlay

Monthly-refreshed portfolio dashboard. Renders as a static HTML page hosted on
GitHub Pages, regenerated by a scheduled GitHub Actions workflow on the first
day of each month.

**Live dashboard:** https://YOUR-USERNAME.github.io/cma-dashboard/

## What this is

- **Strategy:** Barbell Permanent Portfolio (25 SPY / 10 TLT / 15 SHY / 25 GLD /
  25 TIP) with an HYG tactical overlay stealing from TIP based on a credit
  signal (`vol_6m(HYG) + sqrt(T10Y2Y)`).
- **CMA:** TimesFM 2.5 probabilistic forecasts, medians recentered on historical
  6M mean, backtested calibration 83% coverage at the 80% PI level.
- **Audit trail:** every monthly run commits refreshed artifacts to `docs/`;
  `docs/history.csv` accumulates signal readings + weights per month forever.

Full research log, rejected hypotheses, and sanity-check notes: see
[`gep_eval/README.md`](https://github.com/…/CEF/) in the parent research repo.

## Running locally

```bash
pip install --index-url https://download.pytorch.org/whl/cpu torch
pip install -r requirements.txt
python src/dashboard.py
open docs/index.html
```

First run downloads ~800 MB of TimesFM weights to `~/.cache/huggingface/`.
Subsequent runs reuse them.

## Deploy checklist

1. Push this directory to a new GitHub repo.
2. Settings → Pages → Source: "Deploy from a branch" → Branch `main`, folder `/docs`.
3. Settings → Actions → General → Workflow permissions: **Read and write** (so the
   scheduled job can commit the refreshed docs back).
4. Optionally: Actions tab → run workflow manually once to bootstrap docs/.

## Notes & caveats

- **Public repo is fine** — strategy is based on public research and standard risk
  metrics; nothing proprietary is exposed. Private repos work but require GitHub
  Pro for Pages.
- **First scheduled run** takes ~8 min (downloads TimesFM weights). Cached runs:
  ~5 min.
- The credit-signal tactical overlay backtested at t=1.11 — treat as a very
  small, low-confidence tilt on the barbell base, not a real edge. It's included
  in the dashboard because the math is sound even if the out-of-sample
  significance isn't.
